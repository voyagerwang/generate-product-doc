# 输入目录约定（建议）

## 最小可用

- `screenshots/`：页面截图（按页面命名）
- `scope.md`：本次覆盖页面范围

## 推荐

- `examples/`：历史高质量文档（建议 3-10 篇）
- `rules.md`：硬规则（优先级最高）
- `glossary.md`：术语/缩略词定义
- `prototype/`：原型源码或页面链接
- `flows/`：已有流程图素材（可选）

## rules.md 建议字段

### 1) 文档结构
- 是否完整 PRD
- 是否固定 `4.2 功能清单`
- 是否保留/删除总结章节
- 是否需要单列“需求范围说明”（默认不单列，避免与功能清单重复）
- 截图缺失时是阻断定稿还是允许草稿版输出

### 2) 表格结构
- 功能清单表头（建议：平台/一级功能/二级功能/优先级/说明）
- 功能详设表头（建议：功能模块/元素/逻辑说明）
- 是否每页单独表格
- 列宽比例（如 20/40/40）

### 3) 功能清单颗粒度
- 是否只允许模块级能力进入 `4.2`
- 哪些动作必须并回说明列（如上架/下架/删除/导出）
- 哪些占位模块要压缩为汇总说明

### 4) 页面模板
- 列表页模块顺序
- 表单页模块顺序
- 详情页模块顺序
- 占位页是否允许独立成章

### 5) 元素写法
- 字段句式：`字段名，字段类型，字段要求`
- 字段顺序：从上到下、从左到右
- 列表模块是否固定写：展示范围/展示顺序/字段说明/字段交互
- 通用元素（面包屑、标题、返回按钮）是否压缩描述

### 6) 逻辑写法
- 强制覆盖：入参校验、状态流转、计算口径、异常提示
- 复杂字段是否要求阈值/并列/边界规则
- 禁止纯技术实现细节
- 禁止“用于标识/用于识别”类空话

### 7) 字段确定性
- 枚举值是否必须展开
- 若枚举值不明，是否列为待补充项
- 搜索/联想/模糊匹配/精确匹配是否必须说明

### 8) 流程图策略
- 哪些模块必须补流程图
- 流程图放置位置（截图后或模块后）
- 渲染优先级（飞书组件 > 图片 > 文本兜底）

### 9) 交付策略
- 输出目标（飞书或本地 `docx`）
- 飞书输出是否要求显式提供文档链接/ID
- 飞书配置缺失时是否阻断（建议：阻断）
- 更新方式（局部更新优先；仅用户明确要求时整文重写）
- 权限要求（是否“链接可编辑”）

## 默认交付行为

1. 优先先确认输出目标：飞书文档或本地 `docx`。
2. 选择飞书文档时：必须提供飞书链接/ID，并完成 `FEISHU_APP_ID`、`FEISHU_APP_SECRET` 配置。
3. 若飞书模式缺少链接/ID或缺少飞书配置：先阻断并提示补充，不自动降级为 `md`。
4. 选择本地输出时：默认写本地 `docx`。
5. 仅当 `docx` 生成能力不可用时：降级为本地 `md`，并返回降级说明。

## 待补充信息清单（建议输出）

以下信息缺失时，建议统一列入“待补充信息”：
1. 页面截图
2. 枚举值
3. 状态取值
4. 权限范围
5. 统计口径
6. 历史数据处理方式
